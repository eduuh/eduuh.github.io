Modern vim

Vim is ubiquitous. It runs on most computers, and many systems have vim installeg by default. Knowing vim is a skill set you can take with you wherever you go, and fo many people , that’s vim’s killer feature.

An an open source project vim is in good health. Bug fixes and new features are being developed continuously. On many systems, the version of vim that’s installed by default is out of date, but you can enjoy the latest features by upgrading to vim 8.

Version 8 of vim was released in September 2016. It introduced some new features that am writing about in this post.

What is NeoVim

Neovim is a community-run fork of vim that can be used as a drop-in replacemenfor vim. It supports all of the same features vim 8 offers and more. Therefore vim 8 and neovim are interchangeable.

The main goal ares:

to make the editor more hackable.
Introducing a new plugin architucture
sharing responsibility for the codebase with the community.

Vim 8 and Neovim follow different conventios on where to keep their configuration files. Vim 8 typically place them in the ~/.vim directory. whereas Neovim uses the ~/.config/nvim directory. This are important details, but it would get distracting if I mentioned them every time I referenced a runtime file.

Some vim’s commands are formed by combining chords and keystrokes in sequence, and this notation handles them well. consider these examples.

Placeholders

Many of vim’s commands require two or more keystrokes to be entered in sequence. Some commands must be followed by a paticular kind of keystroke, while other commands can be followed by any key on the keyboard. I use curly braces to denote the sed of valid keystrokes that can follow a command.

Here are some examples.

Notation meaning
f{char} Press f, followed by any other character
`{a-z} Press `, followed by any lowercase letter.
m{a-zA-Z} press m, followed by any lowercase or uppercase letters
d{motion} press d, followed by any motion command
{register} while holding ctrl press r, then release ctrl and press the address of a register.
{nondigit} while holding ctrl press v, then release ctrl and press any nondigit key.

Interating with the command Line

Inside of vim , pressing the : key swithes from Normal mode to Command-line mode. In this mode, you can type out Ex commands such as :write and :quit, using the key to execute the command.

The : prompt indicate that the command are to be executed using vim command line mode:

 :s/cool/awesome/g
 :write

Any time you see an Ex command listed inline, such as :write, assume that the key is pressed to execute the command. Nothing happens otherwise so consider to be implicit.

In neovim you can run a shell inside of a terminal buffer, using the :terminal command.

The >> promt indicate that the commands are to be executed in a shell within a terminal buffer:

>> cat readme.md
>> top
Summary of all the prompts available.

| Prompt | Meaning | | : | Use Command-line mode to execute an Ex command| | $ | enter the command line in an external shell | | >> | Enter the command line in an internal shell (within a terminal buffer)|

Installing Plugins to Neovim

Plugins adds new functionalit to vim. You’ll use plugins throughout this post.you need to be able to install them in your command line. Historically vim’s built-in support for installing plugins has been poor, so the vim communinity responded by creating plugin managers. However, since version 8, vim’s package feature has made it easy to install plugins without relying on a plugin manager.

Understanding Script, plugins and packages.

Packages are new features in vim8 that make it easy to mange your plugins. A package is a directory that contains one or more plugins. In turn a plugin is a directory that contains one or more scripts, and a script is a standalone file containing instructions written in Vim Script.

plugins Make it Easy to Organize and Share Scripts

If you write a script that could be useful to other vim Users, you might like to turn it into a plugin. That simply means creating a directory with the name you want to give your plugin, then moving your script into a plugin subdirectory within.

Vim has conventions on how the subdirectories within a plugin should be named. Depending on what a does, it might contains scripts within subdirectories named ftplugin, indent , syntax , among others. when a plugin is installed, Vim automatically sources the script it find in these subdirectories.

Installing a plugin means adding it ot vims ‘runtimepath’

:set runtimepath+=$VIMCONFIG/arbitrary/demo-plugin

Packages Organize and Load Your Plugins

A package is a directory that contains one or more plugins. By conventions, you create package with a $VIMCONFIG/pack directory. Your package should contain a subdirectory called start, which is where you install the plugin that you want to load when vim starts up.

You can create as many packages as you like. you might create a plugin call it bundle where you install plugin written by other people.

You might create another package called myplugins where you keep the plugins that you maintain by yourself.

when vim launches , is searches for plugins under the $VIMCONFIF/pack/*/start/ directory. Any plugin found there are addeg to theruntimepath`.

Indexing the Documentation for installed Plugins

When you add a plugin to one of your packages, all you need to do to start using that plugin is restart vim.But if you want to consult the documentation for a newly installed plugin, you have to do more thing: index its documentation . You do this with the :helptags command.

Vim’s documentation is written is a plaintext format that includes simple markup for defining anchor and hyperlinks .

:help user-manual

Installing an Optional plugin

Some plugin are for everyday use, while others may come in hangy only occasionally. Vim’s package can contain optional plugins, which you can load when needed. Let’s install the Scriptease plugin this way. this plugin provides extra functionality that is useful whel developing vim script files.

Start by creating an opt subdirectory in the bundle package, and then clone the plugin to that directory:

mkdir -p $VIMCONFIG/pack/bundle/opt
cd !$
git clone https://github.com/tpope/vim-scriptease.git

The optinal package is loaded with the :packadd and it will be available lntil vim quits. Next time you launch vim, the opitonal plugin won’t be loaded.

:packadd vim-scriptease

Updating plugins in Your package

plugins, like all software are never complete. there will be bug fixes, new features and general imporvements. If you want to run the latest version of a plugin, you’ll nee to update if from time to time.

Installing plugin with Git means that you can easily update to the latest version by running git pull:

$ cd ~/.config/nvim/pack/start/vim-unimpaired
$ git pull
$ cd ~/.config/nvim/pack/bundle/opt/vim-scriptease
$ git pull

As you install more plugin , the prospect of doing this by hang for each plugin become unappealing. You could automate the process by wirting a shell script.

Opening Files

Opening files is one of the most common tasks you perform when coding. vim’s build-in functionality give you basic commands for opening any readable file by specifyilg its filepath. Tab-completion can save you from typing every character of a filepath, but you may still eng up using a lot of keystrokes to locate the files you want.

Finding Files using Fuzzy Path Matching

You can rely on Vim’s built-in :edit {filepath} command for opening any file on your file system. We are going to untilize a fuzzy finder to reduce the number of keys typed.

Preperations

Vim users are spoild for choice whenn it come to fuzzy finder plugins. In this we will use Junegunn choi’s exceleend fzf tool.

This consist of two parts: a standalon fzf program, which you can run in your shell and a vim plugin, which depends upon the external program.

Installing and Configuiring fzf

You can instal the fzf repository to your bundle package like this:

$ cd ~/.config/nvim/pack/bundle/start
# git clone https://github.com/junegunn/fzf

Not only does this repository include a simple vim plugin (which you’ve now installed but is also contains source code for building the fzf executable. Run this command to install the executable in the repository bin directory.

$ ~/config/nvim/pack/bundle/start/fzf/install -bin
