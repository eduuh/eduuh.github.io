Dotnet core

1. Ominisharp Not Starting

Ominisharp provides crossplatform .Net development. Yes i was using Linux on this one. OmniSharp is a family of Open Source projects, each with one goal: To enable a great .NET experience in YOUR editor of choice Ominsharp is a c# server that analyses your c# code when you are using visual studio code. Without it running, means you cant get real time errors in your code and even warnings.

Thes c# Extension we install to Visual Studio code installed Ominisharp and some other tools.

Error Description:

*The system cannot find the path specified. [ERROR] Error: OmniSharp server load timed out. Use the 'omnisharp.projectLoadTimeout' setting to override the default delay (one minute).*

solution

The solution to this error was to remove all the line of setting that gave an error. For me Initially i was configuring my visual studio code instance to use neovim intergration in the process i added a configuration line to specify the location of neovim in my ubuntu system. The problem with this line is that neovim did not exist in the path indicated. I had suppress this error, from my editor.

I didnt understand why such error would mess with my Ominisharp installation but after removing all the line that had some sort of problem form my visual studio code setting.json file everything works as expected.

The solution to this trouble was found on this stackoverflow thread

2. Using dotnet-ef tool from the cli

The project i was working on is utilizing clean architecture: the error appeared when trying to drop the database created so that i could recreate it.

dotnet ef database drop -p Persitence/ -s Api/

*Your startup project 'Api' doesn't reference Microsoft.EntityFrameworkCore.Design. This package is required for the Entity Framework Core Tools to work. Ensure your startup project is correct, install the package, and try again.*

Solution. This error arised one i upgraded my project from dotnet 2.2 to dotnet 3.0

Dotnet 3.0 does not preinstall the Micrsoft.EntityFramworkCore.Design tool for use us other version dit out of the box. We need to install dotnet entity framework tool globally .

  1. dotnet tool install --global dotnet-ef --version 3.1.1
  2. Install the package indicate in the error. **Microsoft.EntityFrameworkCore.Design**
